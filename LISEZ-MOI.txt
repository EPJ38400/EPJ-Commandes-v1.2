# 📋 EPJ App — Livraison v10.H

**Date** : 10 mai 2026
**Type** : ajout fonctionnel — Module SMS automatique
**Charte qualité** : Option D appliquée

---

## 🎯 Objet de cette livraison

**Activer l'envoi automatique de SMS via Brevo** pour notifier le conducteur de
travaux quand une commande est créée ou modifiée sur un de ses chantiers.

L'app n'envoie pas elle-même les SMS : elle écrit dans une **queue Firestore**
(`smsQueue`), que **Make** lit toutes les minutes pour appeler **Brevo** qui
envoie réellement le SMS.

---

## 📋 Périmètre fonctionnel

### 🟢 Inclus dans cette livraison v10.H

- **Module Commandes** : SMS au conducteur quand une commande est créée OU modifiée
  - Création : si demandeur sans `directAchat` (= statut "En attente de validation")
  - Modification : si la commande repasse en "En attente de validation"
  - **Pas de SMS** si Admin/Direction crée/modifie (ils sont en directAchat=true)
- **Templates SMS éditables** depuis Admin → Modèles SMS
- **Kill switch par template** : tu peux désactiver un type de SMS sans toucher au code

### 🟡 Préparé mais à activer ultérieurement (cron Make quotidien)

- **Module Parc machines** : SMS si outil en retard
- **Module Avancement** : SMS rappel validation mensuelle
- **Module Réserves** : SMS si réserve en retard

Les **templates SMS sont prêts**, le code Firestore aussi. Il manque juste le
2e scénario Make "cron quotidien" à configurer côté Make. Procédure dans
`PROCEDURE_MAKE_SMS.md` (étape 5).

### ❌ Volontairement exclu

- Pas de SMS au demandeur (il consulte son historique dans l'app)
- Pas de SMS à la Direction "X commandes à envoyer aux achats"
  (l'automatisation Esabora qui suit règlera ce flux)
- Pas de quiet hours (envoi 24/7)
- Pas de log des SMS envoyés (économie Firestore — choix Pierre-Julien)

## 🚀 Procédure de déploiement

### Étape 1 — Pousser le code sur GitHub

1. Décompresse `EPJ_app_v10H.zip`
2. GitHub → Add file → Upload files → drag-and-drop
3. Commit : `v10.H — Module SMS automatique via Brevo + Make`
4. Vercel redéploie (~2 min)

### Étape 2 — Importer les nouveaux templates dans l'app

1. Connecte-toi en Admin
2. **Admin → 💬 Modèles SMS**
3. Clique **"Importer les modèles par défaut"**
4. Tu vois apparaître les 4 nouveaux templates :
   - `commande_creee`
   - `commande_modifiee`
   - `avancement_rappel_validation`
   - `reserve_retard`
5. Personnalise les textes si tu veux (sinon les valeurs par défaut conviennent)

### Étape 3 — Configurer Brevo + Make

**📖 Suis le fichier `PROCEDURE_MAKE_SMS.md` livré dans le ZIP**.

C'est la grosse partie. Compte ~30 min en tout :
- Création compte Brevo + clé API + sender (5-10 min)
- Création scénario Make (15-20 min)
- Tests (5-10 min)

### Étape 4 — Vérification

- Crée une commande de test avec un compte monteur sur un chantier dont **toi** tu es conducteur
- Tu dois recevoir un SMS dans la minute qui suit
- Si pas de SMS, voir la section "Dépannage" de `PROCEDURE_MAKE_SMS.md`

## 🐛 Risques identifiés et atténuations

### 1. Make tombe en panne ou est désactivé
- **Risque** : les SMS s'accumulent dans `smsQueue` mais ne partent pas
- **Atténuation** : pas de perte de données, tout est dans Firestore. Une fois
  Make réactivé, les SMS partent dans l'ordre.
- **Surveillance** : vérifier `smsQueue` régulièrement (doc en `status:pending`
  depuis > 1h = alerte)

### 2. Brevo refuse un numéro de téléphone
- **Risque** : SMS jamais envoyé
- **Atténuation** : Make passe le doc en `status:failed` avec `errorMessage`,
  on peut le diagnostiquer manuellement. La normalisation côté app
  (`normalizePhone`) accepte les formats français courants.

### 3. Crédit Brevo épuisé
- **Risque** : SMS plus envoyés
- **Atténuation** : Brevo envoie une alerte mail à <20% restant. Recharger
  via leur interface (~10 min).

### 4. Limite d'opérations Make atteinte
- **Risque** : scénario s'arrête mi-mois
- **Atténuation** : à 220 SMS/mois et 4 ops/SMS = 880 ops/mois (sous le seuil
  gratuit de 1000). Si dépassement : passer au plan Core (~10€/mois).

### 5. Fuite de la clé API Brevo
- **Risque** : usage frauduleux par tiers
- **Atténuation** : la clé est dans Make uniquement, jamais dans GitHub. En
  cas de fuite supposée : régénérer la clé dans Brevo, mettre à jour Make.

### 6. Template désactivé par erreur
- **Risque** : plus de SMS d'un certain type sans s'en rendre compte
- **Atténuation** : la console développeur logue un warning quand un template
  est désactivé. À surveiller au démarrage de la livraison.

### 7. Conducteur sans téléphone
- **Risque** : pas de SMS envoyé pour ce conducteur
- **Atténuation** : `queueSms` retourne `{queued: false, reason}` et logue
  un warning. Vérifier que tous les conducteurs ont leur téléphone dans Admin
  → Utilisateurs.

### 8. Coût qui dérape
- **Risque** : bug d'app qui boucle et envoie 1000 SMS d'un coup
- **Atténuation** : surveiller les coûts Brevo. Pas de garde-fou côté code
  (choix Pierre-Julien). En cas d'urgence : désactiver le template via Admin
  → Modèles SMS.
