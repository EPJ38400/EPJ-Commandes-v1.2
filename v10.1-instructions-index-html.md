# Modifications manuelles à faire sur GitHub pour v10.1

## 1. Fichier `index.html` (à la racine du projet, hors dossier `src/`)

### Trouve cette ligne :
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

### Remplace-la par (important : `viewport-fit=cover`) :
```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
<meta name="apple-mobile-web-app-capable" content="yes" />
<meta name="apple-mobile-web-app-status-bar-style" content="default" />
<meta name="theme-color" content="#F5F5F7" />
```

**Explications :**
- `viewport-fit=cover` : dit à iOS qu'on gère nous-même la safe-area
- `apple-mobile-web-app-capable=yes` : active le mode plein écran quand l'app est ajoutée à l'écran d'accueil
- `apple-mobile-web-app-status-bar-style=default` : barre de statut iOS visible en gris clair
- `theme-color` : couleur du thème (doit matcher EPJ.gray50 = `#F5F5F7` pour que la zone du notch soit du même gris que le fond)

## 2. Procédure

1. Sur GitHub, va dans ton repo `EPJ-Commandes-v1.2`
2. Ouvre `index.html` (à la racine, pas dans `src/`)
3. Clique sur le crayon ✏ pour éditer
4. Remplace la ligne viewport comme indiqué ci-dessus
5. Commit ce changement
6. Push → Vercel redéploie
7. **Cmd+Shift+R** sur ton iPhone Safari (ou effacer données de sites)

## Pourquoi ces 4 balises ensemble ?

- Sans `viewport-fit=cover`, iOS laisse automatiquement un margin autour de l'encoche, ce qui fait perdre de la place
- Avec `viewport-fit=cover`, l'app remplit tout l'écran, MAIS on doit soi-même ajouter `padding: env(safe-area-inset-top)` aux barres du haut (ce qu'on a fait dans le ZIP v10.1)
- `theme-color` donne sa couleur à la zone derrière l'encoche et la Dynamic Island (sinon c'est noir pur)
